# Security Policy & Responsible Disclosure

This repository is dedicated to **education, CTF competition, and authorized
security research**. It does not target, and must not be used against, systems
without explicit authorization.

## Responsible disclosure

If, in the course of research, a vulnerability is discovered in a real,
third-party system:

1. **Do not** access more data than necessary to demonstrate the issue.
2. **Do not** publish details before the vendor has had a reasonable time to fix
   it (industry norm: 90 days, or sooner by mutual agreement).
3. Report privately to the vendor's security contact / bug-bounty program.
4. Only publish a writeup once the issue is fixed or disclosure is authorized,
   and with any sensitive specifics redacted.

## Scope of writeups

- Challenges from CTF platforms (designed to be solved).
- Intentionally-vulnerable practice targets (e.g. DVWA, HackTheBox retired boxes
  where writeups are permitted, local labs).
- Systems the author owns.

## What you will NOT find here

- Working exploits aimed at live third-party production systems.
- Real credentials, tokens, customer data, or PII.
- Tooling whose primary purpose is to attack non-consenting targets.
